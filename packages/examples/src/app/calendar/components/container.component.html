• Drag over the calendar to create a multi-day event.
  • Double-click (or Force Touch) to create a normal event.
  • Grab an event and move it around
  • Resize a multi-day event with the handles on the end.

Requires support for CSS grid and display: contents;. (View on caniuse.com). This is probably too fiddly to use on mobile.

Note that this uses a full-on @ngrx/store setup, and almost 1:1 actions to dnd-core events. This reflects the need to broadcast the state changes and computed values to every subscriber, not just the components where drags and drops begin or end.

The design was mostly borrowed from the macOS calendar, and the CSS pattern is from this post by Jonathan Snook .
